# Content Insights
A child theme inherits the design and functionality of a parent theme, while allowing custom changes without modifying the parent theme’s files. This allows the parent theme to be updated without losing customisations and reduces the risk of breaking core functionality.

Classic themes and block themes differ in how they are built and customised. Classic themes use PHP, template files, widgets, and require manual setup such as enqueuing stylesheets. Block themes use blocks and built-in settings to customise layouts such as headers, menus, and templates, and automatically handle stylesheets. Classic themes rely on template tags to display content dynamically, while block themes use blocks for this purpose.

Template files define how content is displayed on a website. The template hierarchy determines which template file is used based on the type of content requested. Page templates can be applied to posts, pages, or custom post types to change their layout and appearance.
